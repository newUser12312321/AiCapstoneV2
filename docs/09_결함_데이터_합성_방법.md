> 실제 불량 기판을 제작하지 않고도 AI 학습에 충분한 결함 이미지를 확보하는 4가지 방법을 설명합니다.

## 방법 ① — 공개 PCB 결함 데이터셋 활용

### 특징
- 이미 라벨링까지 완성된 공개 데이터셋을 그대로 사용
- 내 기판과 외관이 다를 수 있으므로 방법 ②③과 병행 권장

### 주요 공개 데이터셋

| 데이터셋 | 출처 | 결함 유형 | 특징 |
|---|---|---|---|
| **DeepPCB** | GitHub: `tangsanli5201/DeepPCB` | 단선·단락·핀홀·스퍼·누락 등 6종 | 1,500쌍 정상/불량 이미지 |
| **HRIPCB** | Kaggle 검색: `HRIPCB` | 6가지 결함 | 고해상도 |
| **Roboflow Universe** | `universe.roboflow.com` 검색: `PCB defect` | 다양 | YOLO 포맷 즉시 다운로드 |

### 사용 방법

```bash
# Kaggle CLI로 다운로드
pip install kaggle
kaggle datasets download -d akhatova/pcb-defects

# Roboflow: 웹에서 다운로드 후 YOLO 포맷 선택
# https://universe.roboflow.com 에서 "PCB defect" 검색
```

### 주의사항
- 내 기판 외관과 다르므로 직접 촬영한 이미지와 **혼합 학습**을 권장
- 공개 데이터셋만으로는 내 환경에 최적화된 모델이 되기 어려움

---

## 방법 ② — OpenCV 결함 시뮬레이션 (자동 생성)

### 특징
- 정상 기판 사진에 결함을 직접 그려 넣는 Python 스크립트
- 이미지 1장으로 수십~수백 장의 결함 이미지를 자동 생성
- YOLO 라벨(`.txt`)까지 자동으로 생성됨

### 생성 가능한 결함 유형

| 결함 | 설명 | 시뮬레이션 방법 |
|---|---|---|
| **단선 (Trace Open)** | 구리 배선에 갭 발생 | 배경색 타원 + 노이즈 |
| **까짐 (Metal Damage)** | 금속 표면이 벗겨짐 | 불규칙 다각형 + 블러 |
| **핀홀 (Pinhole)** | 솔더 마스크에 미세 구멍 | 작은 원 + 산화 링 |
| **단락 (Short)** | 두 배선이 잘못 연결됨 | 구리색 브릿지 선 |

### 실행 방법

```bash
# 설치 (opencv만 있으면 됨)
pip install opencv-python numpy

# 미리보기 (결함이 어떻게 보이는지 확인)
python edge/tools/defect_simulator.py \
  --mode preview \
  --input ./my_pcb.jpg \
  --save ./preview_result.jpg

# 데이터셋 일괄 생성
# --input: 정상 기판 이미지 폴더
# --output: 결함 이미지 저장 폴더
# --count: 이미지 1장당 생성 수
# --defects: 이미지당 결함 수
python edge/tools/defect_simulator.py \
  --mode generate \
  --input ./good_pcbs/ \
  --output ./synthetic_dataset/ \
  --count 10 \
  --defects 3
```

### 출력 결과 구조

```
synthetic_dataset/
  images/
    pcb_001_defect_000.jpg   ← 결함 합성 이미지
    pcb_001_defect_001.jpg
    ...
  labels/
    pcb_001_defect_000.txt   ← YOLO 라벨 (자동 생성)
    pcb_001_defect_001.txt
    ...
  classes.txt                ← 클래스 목록
  data.yaml                  ← YOLO 학습 설정 파일
```

### 기대 생성량

| 정상 이미지 수 | `--count` 설정 | 생성되는 결함 이미지 수 |
|---|---|---|
| 20장 | 10 | 200장 |
| 50장 | 10 | 500장 |
| 50장 | 20 | 1,000장 |

> YOLO 학습에는 클래스당 최소 **100~200장** 이상 권장

## 방법 ③ — Copy-Paste 합성 (가장 자연스러운 합성)

### 특징
- 인터넷에서 수집한 실제 결함 패치 이미지를 내 기판에 붙임
- **Poisson Blending**으로 경계를 자연스럽게 합성
- 공개 데이터셋(방법①)과 조합하면 효과 극대화

### 준비물

```
결함 패치 폴더 구조:
  patches/
    trace_open/     ← 단선 결함 패치 이미지들 (인터넷 수집 or DeepPCB에서 crop)
    metal_damage/   ← 까짐 패치
    pinhole/        ← 핀홀 패치
    short/          ← 단락 패치
```

### 패치 이미지 수집 방법

**1. DeepPCB 데이터셋에서 결함 영역만 크롭**

```bash
python edge/tools/copy_paste_augment.py crop \
  --image DeepPCB_open_001.jpg \
  --x1 120 --y1 80 --x2 160 --y2 110 \
  --save ./patches/trace_open/patch_001.jpg
```

**2. 직접 촬영한 이미지에서 CVAT 라벨 좌표로 크롭**
- CVAT에서 라벨링 후 바운딩 박스 좌표 확인
- 위 crop 명령어로 패치 추출

### 실행 방법

```bash
python edge/tools/copy_paste_augment.py batch \
  --good ./good_pcbs/ \
  --patches ./patches/ \
  --output ./paste_dataset/ \
  --per-image 2 \
  --count 10
```

### Copy-Paste vs OpenCV 시뮬레이션 비교

| 항목 | OpenCV 시뮬레이션 | Copy-Paste |
|---|---|---|
| 패치 준비 필요 | 불필요 | 필요 |
| 결함 자연스러움 | 중간 | 높음 |
| 생성 속도 | 빠름 | 보통 |
| 추천 사용 시점 | 빠른 프로토타입 | 최종 학습 데이터 |

## 방법 ④ — Stable Diffusion 인페인팅 (AI 생성)

### 특징
- AI가 직접 결함을 그려 넣음 → 가장 자연스러운 결과
- Google Colab (무료 GPU) 환경에서 실행 가능
- 대량 생성에는 시간이 걸림

### Google Colab에서 실행

```python
# ① 필요 패키지 설치
!pip install diffusers transformers torch accelerate Pillow

# ② 모델 로드 (처음 실행 시 약 5분 소요)
from diffusers import StableDiffusionInpaintPipeline
import torch
from PIL import Image, ImageDraw
import numpy as np

pipe = StableDiffusionInpaintPipeline.from_pretrained(
    "runwayml/stable-diffusion-inpainting",
    torch_dtype=torch.float16
).to("cuda")

# ③ 원본 이미지 로드 (512x512으로 리사이즈)
image = Image.open("good_pcb.jpg").resize((512, 512))

# ④ 결함 넣을 위치 마스크 생성 (흰색 = 결함 생성 영역)
mask = Image.new("L", (512, 512), 0)
draw = ImageDraw.Draw(mask)
draw.ellipse([200, 200, 250, 250], fill=255)  # 결함 위치 지정

# ⑤ 인페인팅 실행
result = pipe(
    prompt="PCB circuit board with scratch damage on copper trace, manufacturing defect, close-up",
    negative_prompt="clean, perfect, no defect",
    image=image,
    mask_image=mask,
    num_inference_steps=30,
    guidance_scale=7.5,
).images[0]

result.save("defective_pcb_ai.jpg")
print("✅ AI 결함 이미지 생성 완료")
```

### 여러 위치에 결함 일괄 생성

```python
import random

def generate_defect_batch(pipe, source_image_path, output_dir, count=20):
    """여러 위치에 결함을 생성해 배치 저장"""
    import os
    os.makedirs(output_dir, exist_ok=True)
    
    orig = Image.open(source_image_path).resize((512, 512))
    
    prompts = [
        "PCB copper trace with open circuit defect, solder mask damage",
        "PCB board with scratch on copper layer, manufacturing defect",
        "PCB with pinhole defect in solder mask, microscopic damage",
        "PCB circuit board with short circuit bridge between traces",
    ]
    
    for i in range(count):
        mask = Image.new("L", (512, 512), 0)
        draw = ImageDraw.Draw(mask)
        
        # 랜덤 위치에 결함 마스크
        cx, cy = random.randint(100, 400), random.randint(100, 400)
        r = random.randint(15, 40)
        draw.ellipse([cx - r, cy - r, cx + r, cy + r], fill=255)
        
        prompt = random.choice(prompts)
        result = pipe(
            prompt=prompt,
            image=orig,
            mask_image=mask,
            num_inference_steps=25,
        ).images[0]
        
        result.save(f"{output_dir}/ai_defect_{i:03d}.jpg")
        print(f"  생성 {i+1}/{count} 완료")

generate_defect_batch(pipe, "good_pcb.jpg", "./ai_generated/", count=30)
```

### 주의사항
- 생성된 이미지는 CVAT에서 수동으로 바운딩 박스 라벨링 필요
- 프롬프트를 여러 번 바꿔가며 가장 자연스러운 결과 선택
- 너무 추상적인 결함은 라벨링 후 제외 권장

## 권장 워크플로우 (단계별)

```
STEP 1 — 공개 데이터셋 다운로드 (1시간)
    Kaggle HRIPCB 또는 DeepPCB 다운로드
    → 기본 결함 패턴 학습용 베이스 확보

STEP 2 — 내 기판 정상 이미지 촬영 (30분)
    실제 C922 웹캠으로 20~30장 촬영
    조명 균일, 초점 고정, 다양한 위치/각도

STEP 3 — defect_simulator.py 실행 (10분)
    python edge/tools/defect_simulator.py \
      --mode generate \
      --input ./my_good_pcbs \
      --output ./synthetic_dataset \
      --count 10 --defects 3
    → 20장 × 10 = 200장 자동 생성

STEP 4 — CVAT 품질 검수 (1~2시간)
    시뮬레이션 결함이 이상한 것만 수동 삭제/수정
    라벨 정확도 확인

STEP 5 — 혼합 데이터셋 구성
    공개 데이터 + 시뮬레이션 결함 + AI 생성
    → 클래스당 200장 이상 목표

STEP 6 — Google Colab에서 학습
    docs/05_데이터셋_학습_파이프라인.md 참고
```
