## Windows / Spring Boot

### ❌ `command not found: mvn` 또는 `'mvn'은(는) 내부 명령어가 아닙니다`

```
원인: Maven이 설치되지 않았거나 PATH에 등록되지 않음

해결:
  winget install Apache.Maven
  # 설치 후 PowerShell 재시작
  mvn --version  # 확인
```

### ❌ `Access denied for user 'root'@'localhost'`

```
원인: application.yml의 DB 비밀번호가 실제 MySQL 비밀번호와 다름

해결:
  notepad backend\src\main\resources\application.yml
  → password: ${DB_PASSWORD:실제_비밀번호_입력}
  또는 PowerShell에서:
  $env:DB_PASSWORD="실제_비밀번호"
  → Spring Boot 재시작
```

### ❌ `mysql : ... 이름으로 인식되지 않습니다`

```
원인: MySQL CLI PATH 미등록 (Workbench만 설치된 경우 포함)

해결:
  1) 실제 실행파일 존재 확인
     Get-ChildItem "C:\Program Files\MySQL" -Recurse -Filter mysql.exe

  2) 직접 실행 확인
     & "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe" --version

  3) PATH 등록 (관리자 PowerShell)
     [Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\MySQL\MySQL Server 8.4\bin", "Machine")

  4) PowerShell 재시작 후 확인
     mysql --version
```

### ❌ `MySQL84 서비스를 시작할 수 없습니다 (NET HELPMSG 3534)`

```
원인: 서비스 등록만 되고 datadir/my.ini 초기화가 안 된 상태

해결(관리자 PowerShell):
  cd "C:\Program Files\MySQL\MySQL Server 8.4\bin"
  .\mysqld --remove MySQL84
  New-Item -ItemType Directory -Force "C:\ProgramData\MySQL\MySQL Server 8.4\Data"
  .\mysqld --initialize-insecure --basedir="C:\Program Files\MySQL\MySQL Server 8.4" --datadir="C:\ProgramData\MySQL\MySQL Server 8.4\Data"
  .\mysqld --install MySQL84 --defaults-file="C:\ProgramData\MySQL\MySQL Server 8.4\my.ini"

  # my.ini 생성
  Set-Content -Encoding ASCII "C:\ProgramData\MySQL\MySQL Server 8.4\my.ini" "[mysqld]"
  Add-Content -Encoding ASCII "C:\ProgramData\MySQL\MySQL Server 8.4\my.ini" "basedir=C:/Program Files/MySQL/MySQL Server 8.4"
  Add-Content -Encoding ASCII "C:\ProgramData\MySQL\MySQL Server 8.4\my.ini" "datadir=C:/ProgramData/MySQL/MySQL Server 8.4/Data"
  Add-Content -Encoding ASCII "C:\ProgramData\MySQL\MySQL Server 8.4\my.ini" "port=3306"
  Add-Content -Encoding ASCII "C:\ProgramData\MySQL\MySQL Server 8.4\my.ini" "sql_mode=STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION"

  net start MySQL84
```

### ❌ `Public Key Retrieval is not allowed`

```
원인: MySQL 8.x 인증 방식과 JDBC URL 옵션 불일치

해결:
  application.yml datasource.url에 아래 옵션 포함:
  allowPublicKeyRetrieval=true

  예:
  jdbc:mysql://localhost:3306/inspection_db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
```

### ❌ `Unknown database 'inspection_db'`

```
원인: MySQL에 데이터베이스가 생성되지 않음

해결:
  mysql -u root -p
  CREATE DATABASE inspection_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  EXIT;
  → Spring Boot 재시작
```

### ❌ Spring Boot 시작 후 브라우저에서 8080 응답 없음

```
원인 1: Windows 방화벽이 8080 포트 차단

해결:
  # PowerShell 관리자 권한 실행
  netsh advfirewall firewall add rule name="SpringBoot-8080" dir=in action=allow protocol=TCP localport=8080

원인 2: MySQL이 실행 중이 아님

해결:
  # Windows 서비스에서 MySQL80 시작
  net start MySQL80
```

### ❌ `Table 'inspection_db.inspection_log' doesn't exist`

```
원인: JPA DDL auto 설정 문제 또는 DB 연결 실패 후 테이블 미생성

해결:
  application.yml에서 확인:
    jpa.hibernate.ddl-auto: update  ← 이 값이어야 함 (validate가 아닌 update)
  → Spring Boot 재시작하면 테이블 자동 생성됨
```

## Windows / React

### ❌ `vite: @/ 경로를 찾을 수 없음` 또는 import 오류

```
원인: vite.config.ts의 resolve.alias 설정 누락

해결: frontend/vite.config.ts 확인
  import { fileURLToPath, URL } from 'node:url'
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
  → npm run dev 재시작
```

### ❌ 대시보드에서 API 요청이 8080으로 안 가고 5173으로 감

```
원인: Vite 프록시 설정 문제

해결: vite.config.ts 확인
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
```

### ❌ 대시보드 차트가 빈 상태로 표시됨

```
원인 1: Spring Boot 서버 미실행
  → mvn spring-boot:run 실행

원인 2: DB에 데이터 없음
  → curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/dummy 실행

원인 3: CORS 오류 (F12 → 콘솔 탭에서 빨간 에러 확인)
  → InspectionController.java의 @CrossOrigin 설정 확인
```

### ❌ `npm install` 중 `plain-crypto-js ... 404` 오류

```
원인: axios 최신 버전 해석 문제 또는 오래된 lockfile 캐시

해결:
  cd frontend
  npm install axios@1.7.2
  npm install
```

### ❌ `@tanstack/react-query-devtools` import 에러

```
원인: 의존성 미설치

해결:
  cd frontend
  npm install @tanstack/react-query-devtools
  npm run dev
```

## 라즈베리파이 / Python

### ❌ `command not found: v4l2-ctl`

```
원인: v4l-utils 패키지 미설치

해결:
  sudo apt install -y v4l-utils
```

### ❌ `RuntimeError: 카메라 장치 0를 열 수 없습니다`

```
원인 1: 웹캠이 연결되지 않음
  → USB 케이블 확인, 다른 포트 시도

원인 2: 장치 번호가 0이 아님
  v4l2-ctl --list-devices  # 실제 장치 번호 확인
  .env에서 CAMERA_DEVICE_INDEX=2 등으로 수정

원인 3: 다른 프로세스가 카메라 점유 중
  sudo fuser /dev/video0  # 점유 프로세스 확인
  sudo kill -9 <PID>
```

### ❌ `ModuleNotFoundError: No module named 'cv2'`

```
원인: 가상환경 활성화 안 함 또는 패키지 미설치

해결:
  source .venv/bin/activate  # 반드시 활성화
  pip install opencv-python
```

### ❌ `gpiozero.exc.BadPinFactory` 또는 GPIO 오류

```
원인: 라즈베리파이가 아닌 환경에서 실행하거나 lgpio 미설치

해결 1: 개발 환경이면 .env에서
  ENVIRONMENT=development  ← MockGPIO 자동 사용

해결 2: 라즈베리파이에서 lgpio 설치
  sudo apt install -y python3-lgpio
  pip install lgpio
```

### ❌ FastAPI 서버는 켜지는데 Spring Boot로 전송이 안 됨

```
원인 1: .env의 SERVER_BASE_URL이 잘못된 IP
  → Windows에서 ipconfig로 IP 확인 후 .env 수정
  SERVER_BASE_URL=http://192.168.0.10:8080  ← 실제 PC IP

원인 2: Windows 방화벽이 8080 차단
  → 방화벽 규칙 추가 (02_Windows_설치_실행.md 참고)

원인 3: PC와 라즈베리파이가 다른 네트워크
  → 같은 공유기에 연결되었는지 확인
  ping <라즈베리파이_IP>  # Windows에서 응답 오면 연결됨
```

### ❌ `Connection refused: 192.168.0.10:8080`

```
원인: Spring Boot가 실행 중이지 않음

확인:
  curl http://192.168.0.10:8080/api/inspections  # Windows에서 먼저 테스트
  → 응답이 오면 정상, 거부되면 Spring Boot 재시작 필요
```

### ❌ YOLO 추론이 너무 느림 (5초 이상)

```
원인: 라즈베리파이 5 CPU 추론은 기본적으로 느림

해결 1: ONNX 포맷 사용
  .env에서:
  YOLO_WEIGHTS_PATH=weights/best.onnx

해결 2: 이미지 크기 축소
  yolo_detector.py에서 imgsz=320으로 낮춤 (정확도 감소 감수)

해결 3: YOLOv8n 대신 더 작은 모델 사용은 현재 nano가 최소

참고 속도:
  YOLOv8n CPU (라즈베리파이 5): 약 200~500ms
  ONNX 최적화 후: 약 150~300ms
```

## SSH / 네트워크

### ❌ `ssh: connect to host 192.168.0.25 port 22: Connection refused`

```
원인 1: 라즈베리파이 SSH 서비스 미실행
  → 라즈베리파이에서 직접:
  sudo systemctl start ssh
  sudo systemctl enable ssh

원인 2: IP 주소가 변경됨
  → 라즈베리파이에서 hostname -I 로 현재 IP 확인
  → 공유기 관리 페이지에서 고정 IP 할당 권장
```

### ❌ IP 주소가 매번 바뀌는 문제

```
해결: 공유기 관리 페이지에서 라즈베리파이 MAC 주소에 고정 IP 할당
  1. 공유기 관리 페이지 접속 (보통 192.168.0.1 또는 192.168.1.1)
  2. DHCP 설정 → 고정 IP 할당
  3. 라즈베리파이 MAC 주소 확인: ip link show eth0 | grep ether
  4. 원하는 IP 고정 (예: 192.168.0.25)
```

### ❌ `WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!`

```
원인: 라즈베리파이를 재설치했거나 IP가 다른 기기와 겹침

해결 (Windows PowerShell):
  ssh-keygen -R 192.168.0.25
  ssh pi@192.168.0.25  # 다시 접속
```

## systemd 서비스

### ❌ `systemctl status inspection-edge` 에서 failed 표시

```
# 상세 로그 확인
journalctl -u inspection-edge -n 50 --no-pager

자주 나오는 원인:
  1. 가상환경 경로 오류 → .service 파일에서 ExecStart 경로 확인
  2. .env 파일 없음 → cp .env.example .env 후 설정
  3. 포트 8000이 이미 사용 중 → sudo lsof -i :8000 으로 확인
```

### ❌ 부팅 후 서비스가 자동 시작되지 않음

```
sudo systemctl enable inspection-edge  # 자동 시작 등록
sudo systemctl daemon-reload
sudo reboot  # 재부팅 후 확인
```
