## Phase 2 — 라즈베리파이 단독 확인

```bash
# SSH 접속 후
cd ~/inspection/edge
source .venv/bin/activate
```

### 웹캠

```bash
v4l2-ctl --list-devices
python3 -c "
import cv2
cap = cv2.VideoCapture(0, cv2.CAP_V4L2)
ret, frame = cap.read()
print('캡처:', '✅ 성공' if ret else '❌ 실패', frame.shape if ret else '')
cap.release()
"
```

```
□ v4l2-ctl에서 /dev/video0 장치 인식됨
□ 캡처 성공 + 이미지 shape: (1080, 1920, 3)
```

### GPIO

```bash
python3 -c "
from hardware.gpio_controller import GpioController
import time
gpio = GpioController()
print('PASS 신호 테스트...')
gpio.signal_pass()
time.sleep(1)
print('FAIL 신호 테스트...')
gpio.signal_fail()
gpio.cleanup()
print('✅ GPIO 테스트 완료')
"
```

```
□ PASS 신호: 초록 LED 2초 켜짐 + 비프 1회
□ FAIL 신호: 빨간 LED 점멸 + 비프 3회
```

### YOLO 모델 로드

```bash
python3 -c "
from inference.yolo_detector import YoloDetector
d = YoloDetector()
d.load()
print('✅ 모델 로드 완료')
"
```

```
□ "모델 준비 완료" 또는 "기본 모델로 대체" 로그 출력
□ 오류 없이 로드됨
```

### FastAPI 서버 단독 실행

```bash
python main.py
```

```
□ "초기화 완료 — 검사 대기 중" 로그 출력
□ 포트 8000 응답 확인 (Windows PowerShell에서):
    curl http://<라즈베리파이_IP>:8000/edge/health
    → {"status": "ok", "environment": "production"}
```

## Phase 3 — 전체 연동 확인

> 이 단계에서는 Spring Boot, React, 라즈베리파이 FastAPI가 **모두 실행 중**이어야 합니다.

### 더미 데이터 전송 테스트

```powershell
# Windows PowerShell에서 실행
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/dummy
```

```
□ 응답: {"message": "더미 전송 성공", "server_response": {"id": 1, ...}}
□ Spring Boot 로그에 "[검사 저장 완료] ID: 1" 출력됨
```

### DB 저장 확인

```powershell
curl http://localhost:8080/api/inspections
```

```
□ 배열에 검사 이력 1건 이상 포함됨
□ result 필드가 "PASS" 또는 "FAIL" 값을 가짐
□ defects 배열이 올바르게 포함됨
```

### 대시보드 실시간 반영 확인

```
□ 브라우저 http://localhost:5173 새로고침
□ 상단 StatCard에 숫자 반영됨 (전체 검사 1건 이상)
□ 도넛 차트(PassFailChart) 렌더링됨
□ 이력 테이블에 방금 전송한 데이터 행이 보임
□ 테이블 행 클릭 시 DefectViewer 패널 열림
```

### 5초 자동 갱신 확인

```powershell
# 더미 데이터 5번 연속 전송
for ($i=1; $i -le 5; $i++) {
    curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/dummy
    Start-Sleep -Seconds 2
}
```

```
□ 브라우저 자동 새로고침 없이 5초 후 테이블에 데이터 추가됨
□ 헤더의 "최종 갱신" 시각이 업데이트됨
```

## Phase 4 — 실제 PCB 검사 테스트

> YOLO 가중치(`best.pt`)가 `weights/` 폴더에 있고  
> `.env`에 `ENVIRONMENT=production`으로 설정된 경우

### 수동 검사 트리거

```powershell
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/trigger
```

```
□ 라즈베리파이가 웹캠으로 이미지 캡처
□ YOLO 추론 실행 (로그에 추론 시간 출력)
□ 판정 결과에 따라 GPIO 신호 출력
□ Spring Boot 서버로 결과 전송
□ 대시보드에 결과 반영
```
