# 10. 시연 가이드

> 프로젝트를 발표하거나 시연할 때 이 문서를 참고하세요.
> 모델 학습 완료 전/후 두 가지 시나리오를 모두 다룹니다.

---

## 시연 전 준비 체크리스트

```
□ Windows PC — Spring Boot 실행 중 (포트 8080)
□ Windows PC — React 대시보드 실행 중 (포트 5173)
□ 라즈베리파이 — FastAPI 서버 실행 중 (포트 8000)
□ 라즈베리파이 — 웹캠 연결됨
□ 라즈베리파이 — LED / 부저 배선 완료
□ 브라우저 — http://localhost:5173 대시보드 화면 열어둠
□ 마우스 패드(검은 매트) 위에 기판 올려둠
```

---

## 시나리오 A — 모델 학습 전 시연 (더미 모드)

> 학습된 `best.pt`가 없어도 시스템 전체 흐름을 시연할 수 있습니다.

### 시연 순서

**1단계 — 정상(PASS) 시연**

```powershell
# Windows PowerShell에서 실행
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/demo/pass
```

```
기대 결과:
  라즈베리파이: 초록 LED 켜짐
  대시보드: PASS 1건 추가, 초록 배지 표시
  터미널: 🟢 PASS 시연 전송 완료
```

**2단계 — 결함(FAIL) 시연**

```powershell
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/demo/fail
```

```
기대 결과:
  라즈베리파이: 빨간 LED 점멸 + 부저 3회 울림
  대시보드: FAIL 1건 추가, 빨간 배지 + 결함 위치 표시
  터미널: 🔴 FAIL 시연 전송 완료
```

**3단계 — 자동 연속 검사 시연**

```powershell
# 5초마다 자동 검사 시작
curl -X POST "http://<라즈베리파이_IP>:8000/edge/inspect/auto/start?interval=5"

# 시연 후 중지
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/auto/stop
```

```
기대 결과:
  5초마다 자동으로 검사 결과가 대시보드에 쌓임
  차트(불량률, 트렌드)가 실시간으로 업데이트됨
```

## 시나리오 B — 모델 학습 후 시연 (실제 검사)

> `best.pt`가 `edge/weights/` 폴더에 있고 `.env`에 `ENVIRONMENT=production`인 경우

### 물리적 결함 시뮬레이션 준비물

```
검은 네임펜 (수성 또는 유성)   → 배선 위에 선 긋기 (TRACE_OPEN 흉내)
작은 검은 스티커               → 금속 패드에 부착 (METAL_DAMAGE 흉내)
알코올 솜 또는 지우개           → 시연 후 결함 제거용
```

### 시연 순서

**STEP 1 — 자동 연속 검사 시작**

```powershell
curl -X POST "http://<라즈베리파이_IP>:8000/edge/inspect/auto/start?interval=3"
```

> 이제 3초마다 카메라가 자동으로 촬영하고 결과를 전송합니다.

---

**STEP 2 — 정상 기판 검사 (PASS 확인)**

```
행동: 마우스 패드 위에 정상 기판을 올려놓음

기대 결과:
  라즈베리파이: 초록 LED 켜짐 (짧게)
  대시보드: PASS 누적
  브라우저: 실시간 차트 업데이트
```

---

**STEP 3 — 결함 발생 시연 (FAIL 확인)**

```
행동: 기판을 잠깐 들고 배선 위에 검은 마커로 선 하나 긋기
      다시 마우스 패드 위에 올려놓기

기대 결과:
  라즈베리파이: 빨간 LED 점멸 + 부저 울림  ← 청중이 직접 확인
  대시보드: FAIL 추가 + 결함 위치 바운딩 박스 표시
  불량률 차트: 수치 증가
```

---

**STEP 4 — 결함 제거 후 복구 (PASS 복귀 확인)**

```
행동: 알코올 솜으로 마커 선 지우기
      다시 마우스 패드 위에 올려놓기

기대 결과:
  라즈베리파이: 다시 초록 LED
  대시보드: PASS 복귀
  불량률 차트: 수치 감소
```

---

**STEP 5 — 자동 검사 중지**

```powershell
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/auto/stop
```

---

**STEP 6 — 대시보드에서 전체 결과 확인**

```
브라우저 http://localhost:5173 에서:
  □ 상단 StatCard: 총 검사 수 / PASS 수 / FAIL 수 / 불량률
  □ 도넛 차트: PASS vs FAIL 비율
  □ 막대 차트: 시간대별 검사 트렌드
  □ 이력 테이블: 각 검사 클릭 → 결함 위치 바운딩 박스 확인
```

      │  FAIL → 빨간 LED + 부저 🔔    │

## 자주 묻는 질문 (발표 중 Q&A 대비)

**Q. 실시간으로 얼마나 빠르게 탐지하나요?**
```
A. 추론 시간: YOLOv8n 기준 약 200~500ms (라즈베리파이 5 CPU)
   전체 검사 사이클: 약 1~2초
   ONNX 최적화 시: 약 150~300ms
```

**Q. 어떤 결함을 탐지하나요?**
```
A. 현재 모델: TRACE_OPEN(단선), METAL_DAMAGE(까짐) 2가지
   피듀셜 마크로 기판 정렬 오차도 함께 계산
   각도 오차 허용 범위: 설정값 초과 시 FAIL 처리
```

**Q. 인터넷이 없어도 동작하나요?**
```
A. 네. 라즈베리파이와 Windows PC가 같은 공유기에 연결되어 있으면
   인터넷 없이 로컬 네트워크에서 동작합니다.
```

**Q. 실제 공장에 적용하려면 무엇이 더 필요한가요?**
```
A. 1. 실제 결함 기판으로 추가 학습 (정확도 향상)
   2. 고정 지그(치구) 제작 (기판 위치 고정)
   3. 산업용 조명 설치 (균일한 조명 확보)
   4. 서버 분리 배포 (라즈베리파이 → 전용 서버)
```

## 긴급 상황 대처

```
부저가 안 울릴 때:
  → GPIO 배선 확인 (06_하드웨어_준비물.md)
  → .env의 ENVIRONMENT=production 확인

대시보드에 데이터가 안 나올 때:
  → Spring Boot 실행 여부 확인 (curl http://localhost:8080/api/inspections)
  → 브라우저 F12 → Console 탭 오류 확인

라즈베리파이 응답이 없을 때:
  → curl http://<IP>:8000/edge/health 로 상태 확인
  → sudo systemctl status inspection-edge

더 자세한 오류 해결 → 08_트러블슈팅.md
```
