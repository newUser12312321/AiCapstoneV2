# 01. 프로젝트 구조 및 아키텍처

## 시스템 개요

라즈베리파이 5 기반의 "탁상형 엣지 비전 검사 스테이션(Desktop Edge Vision Inspection Station)"  
라즈베리파이(Edge)에서 웹캠으로 PCB를 촬영하고 YOLO 모델로 피듀셜 마크 및 결함을 탐지한 뒤,  
판정 결과를 중앙 서버(Windows PC)로 전송하여 React 웹 대시보드에 시각화하는 Two-Tier 아키텍처

## 전체 데이터 흐름

```
[Raspberry Pi 5]
  C922 웹캠 캡처 (1080p)
      ↓ v4l2-ctl (오토포커스 OFF)
  Stage 1: YOLO → 피듀셜 마크 탐지 → 정렬 각도 계산
      ↓ 오차 초과 시 즉시 FAIL
  Stage 2: ROI 크롭 → YOLO → 결함(단선/까짐) 탐지
      ↓
  GPIO → 부저/LED 즉시 알람
      ↓ POST /api/inspections (JSON)
[Windows PC - Spring Boot :8080]
  InspectionLog → MySQL 저장
      ↓ GET /api/inspections (5초 폴링)
[Windows PC - React :5173]
  통계 차트 + 이력 테이블 + 바운딩박스 오버레이
      ↓ (/captures 프록시)
[Raspberry Pi - FastAPI :8000]
  캡처 이미지 정적 서빙 (/captures/*.jpg)
```

## 포트 정리

| 서비스 | 포트 | 실행 위치 |
|---|---|---|
| Python FastAPI (Edge API) | 8000 | 라즈베리파이 |
| Spring Boot (Backend) | 8080 | Windows PC |
| React Vite (Frontend) | 5173 | Windows PC |
| MySQL | 3306 | Windows PC |

## REST API 엔드포인트

### Spring Boot (:8080)
| Method | URL | 설명 |
|---|---|---|
| POST | /api/inspections | 라즈베리파이 검사 결과 수신 |
| GET | /api/inspections | 전체 이력 조회 |
| GET | /api/inspections/{id} | 단건 상세 조회 |
| GET | /api/inspections/recent?limit=10 | 최근 N건 조회 |
| GET | /api/inspections/stats | 통계 요약 |
| GET | /api/inspections/period?from=&to= | 기간 필터 조회 |

### FastAPI (:8000)
| Method | URL | 설명 |
|---|---|---|
| GET | /edge/health | 헬스체크 |
| GET | /edge/status | 카메라/모델 상태 조회 |
| POST | /edge/inspect/trigger | 수동 검사 트리거 |
| POST | /edge/inspect/dummy | 더미 데이터 전송 테스트 |
| GET | /captures/{filename} | 검사 캡처 이미지 조회 |
