### 수집 목표 및 촬영 조건

| 항목 | 권장값 | 이유 |
|---|---|---|
| 정상(PASS) 기판 | **30~50장** | Augmentation으로 150장+ 효과, 중복 촬영 불필요 |
| 결함(FAIL) 기판 | 50~100장 | 결함 유형이 다양하므로 더 많이 필요 |
| 총 원본 이미지 | 80~150장 | 합성 도구로 부족분 보완 |
| 해상도 | 1920×1080 | |
| 카메라 위치 | **실제 검사 위치에 고정** | 운영 환경과 동일해야 함 |
| 기판 위치 변화 | **±3~5mm 범위로 약간씩 달리** | 실제 작업 시 매번 완벽히 같지 않음 |
| 촬영 대상 | 가능하면 **여러 장의 동일 모델 기판** | 기판마다 미세한 차이 반영 |
| 조명 | 실제 운영 조명과 동일하게 고정 | 다양한 조명 촬영 불필요 |
| 초점 | `focus_absolute=30` 고정 (v4l2-ctl) | |
| 각도 | 정면 수직 (틸트 없게) | |
| 배경 | 검은 매트 또는 흰 단색 배경 | |

> **핵심 원칙:** 카메라·조명·각도는 고정, 기판 위치만 약간씩 변화
> 동일한 구도로 100장 반복 촬영은 불필요 — 실질 정보가 늘어나지 않음

### 정상 이미지 촬영 체크리스트

```
□ 카메라와 조명을 실제 검사 환경과 동일하게 세팅
□ 같은 PCB를 약간씩 다른 위치에 놓고 촬영
    ← 좌 3mm  |  정중앙  |  우 3mm  |  위 3mm  |  아래 3mm
□ 가능하면 여러 장(기판 개체)으로 촬영
    PCB-A 10장 + PCB-B 10장 + PCB-C 10장 = 30장
□ 30~50장 완료 후 나머지는 Augmentation으로 보완
```

### 이동·회전 범위 및 구도 기준

**이동/회전 허용 범위**

```
카메라 (고정)
    │
    ↓

┌─────────────────────────────┐  ← 카메라 화면 영역
│                             │
│      ↑ 위로 3~5mm           │
│   ┌──────────┐              │
│ ←│  PCB 기판 │→ 좌우 3~5mm  │
│   └──────────┘              │
│      ↓ 아래로 3~5mm         │
│                             │
│  ↻ 회전 ±3~5도 이내          │
│                             │
└─────────────────────────────┘

기판 전체가 항상 화면 안에 있어야 함 ✅
```

> 이 범위는 실제 작업자가 기판을 올려놓을 때 발생하는 자연스러운 오차입니다.
> 이 범위를 벗어나면 피듀셜 마크 정렬 알고리즘이 보정하므로, 학습 데이터는 이 범위 내의 변화만 다루면 됩니다.

**30장 촬영 순서 예시**

| 촬영 순번 | 기판 상태 |
|---|---|
| 1~5장 | 정중앙 (기준 위치) |
| 6~10장 | 좌로 3mm |
| 11~15장 | 우로 3mm |
| 16~20장 | 위로 3mm |
| 21~25장 | 아래로 3mm |
| 26~30장 | 시계방향 3~5도 회전 |

**기판 전체가 반드시 화면 안에 있어야 하는 이유**

```
❌ 기판 일부가 화면 밖으로 나감
   → 피듀셜 마크가 잘림
   → Stage 1 정렬 실패 → 검사 불가

✅ 기판 전체 + 여백이 화면 안에 있음
   → 피듀셜 마크 4개 모두 인식 가능
   → 정상 검사 동작
```

**카메라 높이 설정 팁**

```
기판이 화면을 꽉 채우면 조금만 이동해도 잘릴 수 있습니다.

권장: 기판이 화면의 70~80%를 차지하도록 카메라 높이 조정
      → 상하좌우 10~15% 여백 확보
      → 5mm 이동해도 기판 전체가 항상 화면 안에 유지
```

## 1.5단계 — Augmentation으로 이미지 수 보완

> 정상 이미지 30~50장만 촬영해도 Augmentation으로 **150~300장 효과**를 낼 수 있습니다.
> Augmentation은 **기존 이미지를 변형해서 새 이미지처럼 사용**하는 기법입니다.

### 방법 1 — YOLO 학습 시 자동 Augmentation (가장 간편)

YOLOv8은 학습 중에 Augmentation을 **자동으로 적용**합니다.  
별도 작업 없이 `model.train()`을 실행하면 다음이 자동 적용됩니다.

```
YOLOv8 기본 자동 Augmentation 항목:
  ✅ 수평 플립 (좌우 반전)
  ✅ 밝기·대비·채도 랜덤 변화 (HSV)
  ✅ 약간의 회전·이동·확대축소
  ✅ Mosaic (이미지 4장을 합쳐 새 이미지 생성)
  ✅ Mixup (두 이미지를 반투명하게 합성)
```

→ **추가 코드 없이 자동으로 데이터를 늘려줍니다.**

### 방법 2 — Albumentations로 직접 Augmentation (더 세밀한 제어)

직접 이미지를 변형해서 파일로 저장하는 방법입니다.  
학습 전에 이미지 수를 실제로 늘려두고 싶을 때 사용합니다.

```bash
pip install albumentations
```

```python
# augment_images.py
# 정상 이미지 30장 → 변형해서 150장으로 늘리기

import cv2
import albumentations as A
from pathlib import Path
import random

# 적용할 변형 목록 (PCB 고정 카메라 환경에 맞게 설정)
transform = A.Compose([
    A.HorizontalFlip(p=0.5),                         # 수평 플립
    A.VerticalFlip(p=0.3),                           # 수직 플립
    A.RandomRotate90(p=0.3),                         # 90도 회전
    A.Rotate(limit=5, p=0.5),                        # ±5도 이내 미세 회전
    A.ShiftScaleRotate(                              # 이동·확대·회전 복합
        shift_limit=0.05,                            # ±5% 이내 이동
        scale_limit=0.05,                            # ±5% 이내 확대축소
        rotate_limit=5,                              # ±5도 이내 회전
        p=0.5
    ),
    A.RandomBrightnessContrast(                      # 밝기·대비 변화
        brightness_limit=0.15,
        contrast_limit=0.15,
        p=0.6
    ),
    A.GaussNoise(var_limit=(5, 20), p=0.3),          # 미세 노이즈 추가
    A.GaussianBlur(blur_limit=(3, 5), p=0.2),        # 약한 블러
    A.CLAHE(clip_limit=2.0, p=0.2),                  # 조명 균일화
])


def augment_folder(input_dir: str, output_dir: str, multiply: int = 5):
    """
    폴더 내 이미지를 Augmentation해서 output_dir에 저장합니다.

    Args:
        input_dir:  원본 이미지 폴더
        output_dir: 저장 폴더
        multiply:   이미지 1장당 생성할 변형 수 (30장 × 5 = 150장)
    """
    src = Path(input_dir)
    dst = Path(output_dir)
    dst.mkdir(parents=True, exist_ok=True)

    images = list(src.glob("*.jpg")) + list(src.glob("*.png"))
    if not images:
        print(f"❌ {input_dir} 에서 이미지를 찾을 수 없습니다.")
        return

    total = 0
    for img_path in images:
        img = cv2.imread(str(img_path))
        img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)

        # 원본도 그대로 복사
        cv2.imwrite(str(dst / img_path.name), cv2.cvtColor(img, cv2.COLOR_RGB2BGR))

        for i in range(multiply):
            augmented = transform(image=img)["image"]
            save_name = f"{img_path.stem}_aug_{i:03d}{img_path.suffix}"
            cv2.imwrite(str(dst / save_name), cv2.cvtColor(augmented, cv2.COLOR_RGB2BGR))
            total += 1

    print(f"✅ Augmentation 완료: {len(images)}장 → {len(images) + total}장 → {output_dir}")


if __name__ == "__main__":
    # 정상 이미지 30장 → 180장으로 늘리기 (원본 30 + 변형 150)
    augment_folder(
        input_dir  = "./dataset/good_pcbs",
        output_dir = "./dataset/augmented_good",
        multiply   = 5,
    )
```

```bash

```

### 방법 1 vs 방법 2 비교

| 항목 | 방법 1 (YOLO 자동) | 방법 2 (Albumentations) |
|---|---|---|
| 추가 코드 | 불필요 | 필요 |
| 파일 실제 생성 | 안 됨 (학습 중에만 적용) | 파일로 저장됨 |
| 사용 시점 | Colab 학습 시 자동 | 학습 전 미리 생성 |
| 권장 상황 | 이미지가 30장 이상일 때 | 이미지가 20장 미만으로 부족할 때 |

> **결론:** 이미지가 30장 이상이면 **방법 1(YOLO 자동)만으로 충분**합니다.
> 이미지가 20장 이하로 너무 적을 때만 방법 2를 사용하세요.

## 2.5단계 — 정상 이미지 + 결함 이미지 데이터셋 병합

> YOLO는 **"이미지 안에서 결함 위치를 찾는"** 모델입니다.
> 정상 이미지(결함 없음)와 결함 이미지를 **함께** 학습해야 오탐(False Positive)을 줄일 수 있습니다.

### 왜 정상 이미지가 필요한가?

```
정상 이미지 없이 학습 → 모든 기판에서 결함을 찾으려 함 → 오탐 폭발
정상 이미지 포함 학습 → "이건 결함이 아님"을 배움 → 정확도 향상
```

### 권장 데이터 구성 비율

```
전체 학습 데이터 (예: 300장 기준)

  결함 있는 이미지  :  정상 이미지
        70%        :      30%

  ├── 결함 이미지 210장  (CVAT 라벨링 + 합성 도구 생성)
  └── 정상 이미지  90장  (직접 촬영한 정상 기판, 라벨 없음)
```

### 정상 이미지 준비 방법

```bash

# 스페이스바로 저장, Q로 종료
```

**2. 정상 이미지용 빈 라벨 파일 일괄 생성 (맥북 또는 Windows에서 실행)**

```python
# create_empty_labels.py
from pathlib import Path

good_images_dir = Path("./dataset/good_pcbs")   # 정상 이미지 폴더
labels_out_dir  = Path("./dataset/labels/train") # 라벨 출력 폴더

labels_out_dir.mkdir(parents=True, exist_ok=True)

count = 0
for img in list(good_images_dir.glob("*.jpg")) + list(good_images_dir.glob("*.png")):
    label_file = labels_out_dir / (img.stem + ".txt")
    label_file.touch()   # 빈 파일 = "결함 없음" 의미
    count += 1
```

```python
# split_dataset.py
import shutil, random
from pathlib import Path

src_img = Path("./merged_dataset/images/train")
src_lbl = Path("./merged_dataset/labels/train")

val_img = Path("./merged_dataset/images/val")
val_lbl = Path("./merged_dataset/labels/val")
val_img.mkdir(parents=True, exist_ok=True)
val_lbl.mkdir(parents=True, exist_ok=True)

all_images = list(src_img.glob("*.jpg")) + list(src_img.glob("*.png"))
val_count  = int(len(all_images) * 0.2)
val_images = random.sample(all_images, val_count)

for img_path in val_images:
    lbl_path = src_lbl / (img_path.stem + ".txt")
    shutil.move(str(img_path), str(val_img / img_path.name))
    if lbl_path.exists():
        shutil.move(str(lbl_path), str(val_lbl / lbl_path.name))

print(f"✅ val 분리 완료: {val_count}장 → val / {len(all_images)-val_count}장 → train")
```
