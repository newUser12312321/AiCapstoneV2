### 촬영 조건

| 항목 | 권장값 | 이유 |
|---|---|---|
| 촬영 장수 | **30~50장** | Augmentation으로 150장+ 효과, 중복 불필요 |
| 카메라 위치 | **실제 검사 위치에 고정** | 운영 환경과 동일해야 함 |
| 기판 위치 변화 | **±3~5mm 범위로 약간씩 달리** | 실제 작업 시 매번 완벽히 같지 않음 |
| 촬영 대상 | 가능하면 **여러 장의 동일 모델 기판** | 기판마다 미세한 차이 반영 |
| 해상도 | 1920×1080 | |
| 조명 | 실제 운영 조명과 동일하게 고정 | |
| 초점 | `focus_absolute=30` 고정 (v4l2-ctl) | |
| 각도 | 정면 수직 (틸트 없게) | |
| 배경 | 검은 매트 또는 흰 단색 배경 | |

> **핵심 원칙:** 카메라·조명·각도는 고정, 기판 위치만 약간씩 변화
> 동일 구도 100장 반복 촬영은 불필요 — 실질 정보가 늘어나지 않음

### 촬영 체크리스트

```
□ 카메라와 조명을 실제 검사 환경과 동일하게 세팅
□ 기판이 화면의 70~80%를 차지하도록 카메라 높이 조정
    → 상하좌우 10~15% 여백 확보 (이동해도 기판 전체 유지)
□ 같은 PCB를 약간씩 다른 위치에 놓고 촬영
    ← 좌 3mm  |  정중앙  |  우 3mm  |  위 3mm  |  아래 3mm
□ 가능하면 여러 기판 개체로 촬영
    PCB-A 10장 + PCB-B 10장 + PCB-C 10장 = 30장
```

### 이동·회전 범위 기준

```
카메라 (고정)
    │
    ↓

┌─────────────────────────────┐  ← 카메라 화면 영역
│                             │
│      ↑ 위로 3~5mm           │
│   ┌──────────┐              │
│ ←│  PCB 기판 │→ 좌우 3~5mm  │
│   └──────────┘              │
│      ↓ 아래로 3~5mm         │
│                             │
│  ↻ 회전 ±3~5도 이내          │
│                             │
└─────────────────────────────┘

기판 전체가 항상 화면 안에 있어야 함 ✅
```

**30장 촬영 순서 예시**

| 촬영 순번 | 기판 상태 |
|---|---|
| 1~5장 | 정중앙 (기준 위치) |
| 6~10장 | 좌로 3mm |
| 11~15장 | 우로 3mm |
| 16~20장 | 위로 3mm |
| 21~25장 | 아래로 3mm |
| 26~30장 | 시계방향 3~5도 회전 |

```
❌ 기판 일부가 화면 밖으로 나감
   → 피듀셜 마크가 잘림 → Stage 1 정렬 실패 → 검사 불가

✅ 기판 전체 + 여백이 화면 안에 있음
   → 피듀셜 마크 4개 모두 인식 가능 → 정상 검사 동작
```

```
  [●]─────────────────────[●]
   │                        │
   │       PCB 기판          │
   │                        │
  [●]─────────────────────[●]

● = 피듀셜 마크 (보통 4개, 모서리 근처)
```

**기본 단축키:**

| 키 | 동작 |
|---|---|

| `Ctrl+S` | 저장 |
| `Del` | 선택한 박스 삭제 |
| `Tab` | 다음 객체로 이동 |

**AI 보조 라벨링 (SAM — 클릭 한 번으로 자동 경계 인식):**

### 3-3. train / val 분리 (8:2 비율)

```python
# split_dataset.py — 맥북 또는 Windows에서 실행
import shutil, random
from pathlib import Path

src_img = Path("./merged_dataset/images/train")
src_lbl = Path("./merged_dataset/labels/train")

val_img = Path("./merged_dataset/images/val")
val_lbl = Path("./merged_dataset/labels/val")
val_img.mkdir(parents=True, exist_ok=True)
val_lbl.mkdir(parents=True, exist_ok=True)

all_images = list(src_img.glob("*.jpg")) + list(src_img.glob("*.png"))
val_count  = int(len(all_images) * 0.2)
val_images = random.sample(all_images, val_count)

for img_path in val_images:
    lbl_path = src_lbl / (img_path.stem + ".txt")
    shutil.move(str(img_path), str(val_img / img_path.name))
    if lbl_path.exists():
        shutil.move(str(lbl_path), str(val_lbl / lbl_path.name))

print(f"✅ val 분리 완료: {val_count}장 → val / {len(all_images)-val_count}장 → train")
```

**셀 3 — YOLOv8 학습**

```python
from ultralytics import YOLO

# YOLOv8n (nano) — 라즈베리파이5 최적화 경량 모델
model = YOLO('yolov8n.pt')

results = model.train(
    data      = f'{dataset_path}/data.yaml',
    epochs    = 100,
    imgsz     = 640,
    batch     = 16,        # OOM 발생 시 8로 낮춤
    device    = 0,
    project   = '/content/drive/MyDrive/AiCapstone/runs',
    name      = 'pcb_v1',
    patience  = 20,        # 20 epoch 개선 없으면 조기 종료
    save      = True,
    plots     = True,
    # ── PCB 고정 카메라 환경에 맞는 Augmentation 설정 ──
    fliplr    = 0.5,       # 수평 플립
    flipud    = 0.0,       # 수직 플립 (고정 카메라면 0 권장)
    degrees   = 5.0,       # 회전 ±5도
    translate = 0.05,      # 이동 ±5%
    scale     = 0.05,      # 확대축소 ±5%
    mosaic    = 1.0,
)

print("✅ 학습 완료:", results.save_dir)

metrics = best_model.val(data=f'{dataset_path}/data.yaml')

print(f"mAP50:     {metrics.box.map50:.3f}")    # 목표: 0.85 이상
print(f"mAP50-95:  {metrics.box.map:.3f}")
print(f"Precision: {metrics.box.mp:.3f}")
print(f"Recall:    {metrics.box.mr:.3f}")
```
