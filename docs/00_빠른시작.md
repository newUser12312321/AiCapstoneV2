# 00. 빠른 시작 가이드 (Quick Start)

> 이 문서를 처음 읽는다면 여기서 시작하세요.
> 전체 세팅을 **처음부터 끝까지** 순서대로 안내합니다.

---

## 이 프로젝트가 하는 일

```
라즈베리파이 웹캠으로 PCB 기판을 촬영
    ↓
AI(YOLOv8)가 결함(단선·까짐)을 자동으로 탐지
    ↓
불량이면 부저·LED 즉시 알람
    ↓
검사 결과를 Windows PC 서버로 전송
    ↓
웹 대시보드에서 실시간으로 결과 확인
```

---

## 필요한 하드웨어

| 항목 | 필수 여부 | 비고 |
|---|---|---|
| 라즈베리파이 5 (4GB 이상) | 필수 | 라즈비안 OS 설치 |
| microSD 카드 32GB 이상 | 필수 | Class 10 이상 |
| USB 웹캠 (C922 권장) | 필수 | 1080p 지원 필수 |
| Windows 데스크탑 또는 노트북 | 필수 | 서버 + 대시보드 실행용 |
| 공유기 (라우터) | 필수 | PC와 라즈베리파이 같은 LAN |
| 빨간 LED × 1 | 권장 | FAIL 표시 |
| 초록 LED × 1 | 권장 | PASS 표시 |
| 능동 부저 × 1 | 권장 | FAIL 알람음 |
| 220Ω 저항 × 2 | 권장 | LED 보호용 |
| 점퍼 와이어 | 권장 | GPIO 연결용 |

## 전체 세팅 순서 (처음 한 번만)

### Windows PC 작업

```
Step 1. 소프트웨어 설치 (위 표 참고)
Step 2. git clone https://github.com/newUser12312321/AiCapstoneV2.git
Step 3. MySQL에 inspection_db 데이터베이스 생성
Step 4. application.yml에 MySQL 비밀번호 입력
Step 5. 포트 8080 방화벽 허용
```

> 자세한 방법 → [02_Windows_설치_실행.md](02_Windows_설치_실행.md)

### 라즈베리파이 작업

```
Step 6.  SSH 활성화 및 IP 주소 확인
Step 7.  Windows에서 SSH 접속
Step 8.  시스템 패키지 설치 (python3-venv, v4l-utils 등)
Step 9.  GitHub에서 코드 클론
Step 10. pip install -r requirements.txt
Step 11. .env 파일에 Windows PC IP 입력
Step 12. 웹캠 연결 확인
Step 13. GPIO 배선 (LED, 부저)
```

> 자세한 방법 → [03_라즈베리파이_설정.md](03_라즈베리파이_설정.md)

## 매일 실행 순서 (세팅 완료 후)

세팅이 끝났다면 매번 이 순서로 실행합니다.

```
[Windows 터미널 1]
cd C:\Projects\AiCapstoneV2\backend
mvn spring-boot:run
→ "Started InspectionApplication" 메시지 확인

[Windows 터미널 2]
cd C:\Projects\AiCapstoneV2\frontend
npm run dev
→ "Local: http://localhost:5173" 확인

[Windows 터미널 3 — SSH]
ssh pi@<라즈베리파이_IP>
cd ~/inspection/edge && source .venv/bin/activate
python main.py
→ "초기화 완료 — 검사 대기 중" 확인

[브라우저]
http://localhost:5173 접속 → 대시보드 확인
```

## 동작 확인 (3분 테스트)

모든 서버가 켜진 상태에서 PowerShell에서 실행:

```powershell
# 1. 라즈베리파이 서버 살아있는지 확인
curl http://<라즈베리파이_IP>:8000/edge/health
# 기대 응답: {"status": "ok", ...}

# 2. 더미 검사 데이터 전송 테스트
curl -X POST http://<라즈베리파이_IP>:8000/edge/inspect/dummy
# 기대 응답: {"message": "더미 전송 성공", ...}

# 3. DB 저장 확인
curl http://localhost:8080/api/inspections
# 기대 응답: [{"id":1, "result":"PASS" or "FAIL", ...}]

# 4. 브라우저에서 대시보드 확인
# http://localhost:5173 → 차트와 테이블에 데이터 표시
```
